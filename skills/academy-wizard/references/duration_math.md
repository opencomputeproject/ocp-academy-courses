# Duration math

The wizard's first hard decision is how many modules and how many slides per module to propose. Get this right and the rest of the work follows from it.

## Core rule of thumb

- **~2 minutes of seat time per slide** including narration playback and read-and-think time. Title and "Up Next" slides count as half (~1 minute each) because their narration is short.
- **8–10 content slides per module.** This makes each module ~15–20 minutes — the sweet spot for adult attention.
- **Always include**, per module: a title slide, an objectives slide (except module 1, which uses a course-overview slide instead), 4–7 content slides, a key takeaways slide, and an "Up Next" / "Course Complete" slide.

## Suggested shapes by total duration

| Target duration | Modules | Slides per module | Total slides |
|---|---|---|---|
| 15–20 min | 2 | 8 | 16 |
| 25–35 min | 3 | 8–10 | ~27 |
| 40–60 min | 4 | 9–10 | ~38 |
| 60–90 min | 5–6 | 9–10 | ~50–55 |
| 90–120 min | 6–7 | 10–12 | ~70 |
| 2–3 hours | 8–10 | 10–12 | ~100 |

These are *starting points* to propose to the user. They will push back. That's expected.

## When the user wants a different shape

If the user asks for, say, "a 30-minute course with 6 short modules," respect it — short focused modules are valid for reference courses learners dip into. But push back gently if the math doesn't work: "Six modules in 30 minutes is 5 minutes each, which is roughly 3 slides per module. That's tight for technical content. Want me to propose 3 modules of 10 minutes instead, or do you really want the short-burst format?"

## Narration word count

Narration runs at **140–160 words per minute** for technical content. Plan script length accordingly:

| Slide seat-time target | Narration words |
|---|---|
| 30 seconds (title slide) | 70–80 |
| 60 seconds | 140–160 |
| 90 seconds | 210–240 |
| 120 seconds (rich content slide) | 280–320 |

If a slide's narration runs past 320 words, split the slide. Listeners zone out.
