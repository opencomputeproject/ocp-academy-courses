# TTS note: pronounce FEC as "feck"; spell technical acronyms letter by letter where written with spaces.
主機 SDK 和命令列 API 是規範對運維人員的有用之處。規範列出了直接對應到現場工作流程程的類別。

標準的 CMIS 存取允許軟體讀取本地模組狀態。遠端 CMIS 存取允許軟體從遠端端點獲取標準資料。遙測下載會暴露本地或遠端模組引數。事件日誌命令讀取近期事件、清除日誌並調整滯後。韌體命令用於管理本地或遠端更新的工作流程程。

規範還包括紀元時間和模組鏈路評分顯示。這些類別很重要，因為它們連線了證據鏈：當前數值、歷史事件、時間相關性和簡明的健康評分。一個好的API能讓這條鏈條在資格驗證和故障事件中都能使用。
API 類別還應對應到文件和培訓。運維人員應知道哪個命令檢索當前遙測資料，哪個命令檢索近期事件，哪個命令清除歷史記錄，哪個命令改變行為。
