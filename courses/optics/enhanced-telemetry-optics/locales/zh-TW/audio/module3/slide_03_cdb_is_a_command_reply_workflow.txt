# TTS note: pronounce FEC as "feck"; spell technical acronyms letter by letter where written with spaces.
CDB 為主機和模組提供命令-回應模式。規範描述了一個典型的序列，分為四個步驟。

首先，主機透過填寫所需的頭欄位和有效載荷來準備命令訊息。其次，它透過將訊息寫入相應位置來觸發命令。第三，它輪詢CDB狀態暫存器以監控進展。第四，命令完成後，它會檢索回覆頭和有效載荷。

這一模式是本模組的主線。無論主機是在獲取遙測資料、請求事件日誌記錄，還是使用更大的有效載荷區域，工作方式都是相同的：寫入請求，觸發請求，檢查狀態，然後讀取響應。
這四個步驟也讓軟體出現故障時更容易排查。你可以詢問命令是否正確生成，是否觸發，狀態是否被讀取，以及回覆是否對應已完成的命令。
