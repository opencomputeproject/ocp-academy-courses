# TTS note: pronounce FEC as "feck"; spell technical acronyms letter by letter where written with spaces.
本課程基於Credo和Oracle志願者撰寫的OCP社群貢獻。課程分為四個部分。首先，我們構建了當光鏈路跨越交換機埠、收發模組、光纖路徑以及可能位於裸機伺服器內部的遠端端點時所出現的可見性差距。

其次，我們分析讓鏈路狀態可轉化為行動的遙測類別：光學品質、電氣品質、錯誤、恢復行為、溫度、功率和管理上下文。

第三，我們考察在主機軟體和模組之間傳遞這些資訊的 命令-回應 工作流程程。這包括CDB、事件日誌以及遠端CMIS訪問。

最後，我們將遙測模型與韌體管理、主機 API 以及部署準備狀態連線起來。到最後，你應該能解釋運維模型，而無需陷入位元組對映細節。
在你瀏覽模組時請記住一個問題：這些資訊是否能幫助運維人員更快決定下一步行動？如果答案是肯定的，那它就屬於可靠性工作流程程。
