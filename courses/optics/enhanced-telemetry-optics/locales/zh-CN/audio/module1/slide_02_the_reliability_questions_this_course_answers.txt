# TTS note: pronounce FEC as "feck"; spell technical acronyms letter by letter where written with spaces.
本课程基于Credo和Oracle志愿者撰写的OCP社区贡献。课程分为四个部分。首先，我们构建了当光链路跨越交换机端口、收发模块、光纤路径以及可能位于裸机服务器内部的远端端点时所出现的可见性差距。

其次，我们分析让链路状态可转化为行动的遥测类别：光学质量、电气质量、错误、恢复行为、温度、功率和管理上下文。

第三，我们考察在主机软件和模块之间传递这些信息的 命令-响应 工作流程。这包括CDB、事件日志以及远程CMIS访问。

最后，我们将遥测模型与固件管理、主机 API 以及部署准备状态连接起来。到最后，你应该能解释运维模型，而无需陷入字节映射细节。
在你浏览模块时请记住一个问题：这些信息是否能帮助运维人员更快决定下一步行动？如果答案是肯定的，那它就属于可靠性工作流程。
