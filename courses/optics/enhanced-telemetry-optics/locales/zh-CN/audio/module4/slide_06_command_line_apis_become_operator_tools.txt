# TTS note: pronounce FEC as "feck"; spell technical acronyms letter by letter where written with spaces.
主机 SDK 和命令行 API 是规范对运维人员的有用之处。规范列出了直接对应到现场工作流程的类别。

标准的 CMIS 访问允许软件读取本地模块状态。远程 CMIS 访问允许软件从远端端点获取标准数据。遥测下载会暴露本地或远端模块参数。事件日志命令读取近期事件、清除日志并调整滞后。固件命令用于管理本地或远程更新的工作流程。

规范还包括纪元时间和模块链路评分显示。这些类别很重要，因为它们连接了证据链：当前数值、历史事件、时间相关性和简明的健康评分。一个好的API能让这条链条在资格验证和故障事件中都能使用。
API 类别还应对应到文档和培训。运维人员应知道哪个命令检索当前遥测数据，哪个命令检索近期事件，哪个命令清除历史记录，哪个命令改变行为。
