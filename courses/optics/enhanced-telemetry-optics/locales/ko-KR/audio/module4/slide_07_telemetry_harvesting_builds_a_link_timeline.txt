# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
호스트 소프트웨어 섹션은 중요한 의도를 설명합니다: 호스트는 현재 링크 상태를 지속적으로 수집하여 링크 성능 타임라인을 생성하고, 이를 텔레메트리 서버로 스트리밍하여 네트워크 전체 네트워크 맵으로 집계할 수 있습니다.

그 문장이 전체 과정을 연결해줍니다. 단 하나의 쿼리가 유용합니다. 타임라인이 더 낫습니다. 네트워크 맵는 더욱 좋은데, 스위치, 모듈, 고속 레인, 원격 종단점, 시간 등 여러 가지 신호를 비교할 수 있기 때문입니다.

타임라인에는 현재 지표, 이벤트 로그 기록, 원격 파트너 점검, 그리고 사전 펙 B E R 및 스위치 serdes 수신기에서의 정정 불가능한 코드워드 같은 호스트 측 측정값이 포함되어야 합니다. 이 값들이 일치할 때, 문제 해결은 고립된 경보에서 증거 기반 작업으로 전환됩니다.
타임라인은 스냅샷이 놓친 패턴을 드러낼 수 있습니다. 예를 들어, 재연결 전 온도 상승이나 로컬 경보 전 원격 엔드포인트 이벤트는 조사를 추측에서 순서 분석으로 바꿀 수 있습니다.
