# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
이 마지막 모듈은 신뢰성 워크플로우를 연결합니다. 펌웨어 관리는 단순히 이미지 다운로드를 하는 것이 아닙니다. 이는 기능 탐색, 전송, 상태, 활성화, 커밋, 복구 동작을 포함하는 라이프사이클입니다.

호스트 소프트웨어는 여전히 로컬 제어 지점으로 남아 있습니다. 이 장치는 I 제곱 C 위에 있는 로컬 모듈과 통신하고, 로컬 모듈은 원격 파트너 인밴드 데이터를 가져옵니다. 운영자 API는 텔레메트리 다운로드, 이벤트 로그, 원격 C M I S 접근, 펌웨어 업데이트, 에포크 타임, 히스테리시스, 링크 등급 등 일반적인 동작을 노출합니다.

이 요소들이 함께 작동하면 텔레메트리는 타임라인이 되고, 타임라인은 네트워크 전체의 상태 뷰가 됩니다. 이것이 공통 인터페이스의 운영 가치입니다.
이 과정은 운영 목표가 더 넓기 때문에 의도적으로 바이트 수준 이상의 구현을 유지합니다. 학습자들은 링크 신뢰성이 위태로울 때 도구, 벤더, 내부 팀에 더 나은 질문을 할 수 있어야 합니다.
