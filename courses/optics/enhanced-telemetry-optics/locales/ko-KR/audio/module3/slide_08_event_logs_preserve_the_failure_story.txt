# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
현재 텔레메트리는 현재 링크 상태를 알려줍니다. 이벤트 로그가 어떻게 거기에 도달했는지 설명해 줍니다.

이 명세는 비휘발성 메모리에서의 이벤트 로깅을 설명합니다. 또한 고유한 시퀀스 값, 타임스탬프 필드, 이벤트 식별자, 상세 필드를 가진 32바이트 이벤트 로그 항목을 설명합니다. 가장 작은 시퀀스 값은 가장 오래된 레코드를 나타내며, 시퀀스는 랩(wrap)할 수 있습니다.

이 구조는 모듈이 이동, 리셋, 사후에 검사될 때 중요합니다. 로그가 순서, 시간, 이벤트 유형, 관련 세부사항을 보존한다면, 운영자는 대시보드에 처음 도달한 경보에 의존하지 않고 장애 이력을 재구성할 수 있습니다.

수업의 핵심은 간단합니다: 사건 이력은 일시적인 증상을 증거로 만듭니다.
이벤트 기록은 공급업체와 운영자 간의 대화 중에도 유용합니다. 문제가 언제 발생했을지 논쟁하는 대신, 팀은 순서대로 기록, 타임스탬프, 세부 정보를 호스트 로그와 유지보수 창과 비교할 수 있습니다.
