# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
상태 확인은 명령-응답 워크플로우를 정직하게 유지하는 핵심입니다. 모듈이 사용 중일 수 있습니다. 명령이 성공적으로 완료될 수도 있습니다. 명령이 실패할 수도 있습니다. 호스트 소프트웨어는 응답을 유효한 것으로 처리하기 전에 어떤 상태를 보고 있는지 알아야 합니다.

그래서 명세에는 C D B 명령 상태 레지스터와 완료 표시기가 포함되어 있습니다. 중요한 것은 명령을 완성하는 것만이 아닙니다. 핵심은 호스트 행동을 반복 가능하게 만드는 것입니다.

신뢰할 수 있는 운영자 도구는 텔레메트리가 준비되었다고 추측해서는 안 됩니다. 명령어를 트리거하고, 완료를 폴링하며, 실패를 명확히 처리한 후 응답을 가져와야 합니다. 이러한 소프트웨어 규율이 저수준 모듈 동작을 신뢰할 수 있는 운영 증거로 만들 수 있게 합니다.
좋은 상태 동작은 자동화도 향상시킵니다. 스크립트는 안전하게 재시도, 후퇴, 명령 실패 보고, 오래된 데이터를 제공하기 전에 멈출 수 있습니다. 이것이 유용한 도구와 위험한 지름길의 차이입니다.
