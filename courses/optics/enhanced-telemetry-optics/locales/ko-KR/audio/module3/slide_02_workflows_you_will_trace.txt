# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
이 모듈이 끝날 때쯤이면 명령어 메시지 준비, 명령어 트리거, 상태 레지스터 폴링, 응답 가져오기 등 4단계로 이루어진 C D B 워크플로우를 추적할 수 있습니다.

또한 바이트 필드를 외우지 않고도 페이지 9Fh와 확장 페이로드 페이지의 목적을 설명할 수 있습니다.

그 다음 명령어 워크플로우를 이벤트 로그에 연결합니다. 이벤트 로그는 즉각적인 증상이 지나간 후에도 시간, 식별 정보, 이벤트 유형을 보존하기 때문에 중요합니다.

마지막으로, 원격 바이트 접근과 원격 페이지 접근을 구분할 것입니다. 두 방법 모두 로컬 준수 모듈을 통해 원격 엔드포인트의 정보를 전달하는 방법입니다.
워크플로우가 저수준처럼 보일 수 있지만, 펌웨어 엔지니어만을 위한 것이 아닙니다. 운영자는 도구가 올바르게 재시도하거나, 실패를 명확히 보고하거나, 링크가 복구된 후에도 이벤트 기록을 보존할 때 그 영향을 느낍니다.
