# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
이 과정은 Credo와 Oracle의 자원봉사자들이 작성한 OCP 커뮤니티 기여를 바탕으로 합니다. 이 과정은 네 부분으로 구성되어 있습니다. 먼저, 광 링크가 스위치 포트, 트랜시버 모듈, 광섬유 경로, 그리고 베어메탈 서버 내부에 위치한 원격 엔드포인트를 넘날 때 나타나는 가시성 격차를 구상합니다.

둘째, 링크 상태를 실용적으로 만드는 텔레메트리 범주들, 즉 광학적 품질, 전기 품질, 오류, 복구 동작, 온도, 전력, 관리 맥락 등을 다룹니다.

셋째, 이 정보를 호스트 소프트웨어와 모듈 간에 이동시키는 명령-응답 워크플로우를 살펴봅니다. 여기에는 C D B, 이벤트 로그, 원격 C M I S 접근이 포함됩니다.

마지막으로, 텔레메트리 모델을 펌웨어 관리, 호스트 API, 배포 준비와 연결합니다. 끝나면 바이트 맵에 갇히지 않고 운영 모델을 설명할 수 있을 것입니다.
모듈을 진행하면서 한 가지 질문을 기억하세요: 이 정보가 운영자가 다음 행동을 더 빠르게 결정하는 데 도움이 될까요? 만약 '예'라면, 신뢰성 워크플로우에 포함되어야 합니다.
