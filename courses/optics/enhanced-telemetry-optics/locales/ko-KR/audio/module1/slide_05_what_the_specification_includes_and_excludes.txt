# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
스코프는 여기서 유용한 일을 하고 있습니다. 이 명세는 호환 가능한 400G, 800G, 1.6T 트랜시버에 적용되며, 모듈 P M D, 즉 물리-매체 의존적 구현에 독립적으로 적용되도록 설계되었습니다. 즉, 텔레메트리 인터페이스는 단일 광학 구현 세부 사항에 의존해서는 안 됩니다.

이 명세는 또한 OIF의 공통 관리 인터페이스 명세인 C M I S를 기반으로 합니다. 이 기술이 C M I S를 대체하지는 않습니다. 이 기능은 C M I S 기능을 사용하며, 광학 기술 원격 측정이 운영 질문에 보다 일관되게 답변할 수 있도록 특정 정의와 수정을 추가합니다.

제외 조항도 중요합니다. 텔레메트리 서버는 범위 밖에 있습니다. 이로 인해 운영자는 트랜시버 경계의 공통 인터페이스를 약화시키지 않으면서도 모듈과 호스트 데이터를 자체 관측 스택에 통합할 수 있습니다.
스코프 컨트롤은 약점이 아닙니다. 이것이 기여를 실행 가능하게 만드는 요소입니다. 인터페이스를 표준화하면서 상위 서버 설계는 열어두면서, 모델은 하나의 관측 스택을 강요하지 않고 다양한 운영자 환경에 적합할 수 있습니다.
