# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Scope is doing useful work here. The spec applies to compliant 400G, 800G, and 1.6T transceivers, and it is intended to be independent of the module P M D, or physical-media-dependent implementation. In other words, the telemetry interface should not depend on one optical implementation detail.

The spec also builds on C M I S, the Common Management Interface Specification from OIF. It does not replace C M I S. It uses C M I S functions and adds specific definitions and modifications so optics telemetry can answer operational questions more consistently.

The exclusion matters too. The telemetry server is outside scope. That gives operators room to integrate the module and host data into their own observability stack without weakening the common interface at the transceiver boundary.
Scope control is not a weakness. It is what makes the contribution implementable. By standardizing the interface while leaving higher-level server design open, the model can fit different operator environments without forcing one observability stack.
