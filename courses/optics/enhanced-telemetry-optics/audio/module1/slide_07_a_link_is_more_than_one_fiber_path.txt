# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
A key shift is to stop thinking about a link as only the fiber path. The point-to-point figure breaks the connection into monitored segments. On one side, the switch talks electrically to its local module. In the middle, the two modules communicate optically. On the far side, the remote module talks electrically to the host.

That segmentation matters during debug. A poor optical signal does not mean the same thing as a host electrical receiver issue. A local alarm may be reporting a symptom whose cause is on a remote segment.

The telemetry model gives host software a way to ask for evidence across the full path. Once each segment can be measured and reported in a consistent way, operators can reason about where the degradation lives instead of swapping parts by instinct.
This segment view is also useful when symptoms disagree. A host may see one behavior, a module may log another, and the remote partner may show a third. The model gives you places to compare, not just places to blame.
