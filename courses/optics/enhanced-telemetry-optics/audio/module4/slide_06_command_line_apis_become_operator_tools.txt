# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
The host SDK and command-line APIs are where the specification becomes useful to operators. The spec lists categories that map directly to field workflows.

Standard C M I S access lets software read local module state. Remote C M I S access lets software extract standard data from the far endpoint. Telemetry download exposes local or remote module parameters. Event-log commands read recent events, clear logs, and adjust hysteresis. Firmware commands manage local or remote update workflows.

The spec also includes epoch time and module link rating display. Those categories matter because they connect the evidence chain: current values, historical events, time correlation, and a concise health score. A good API makes that chain usable during qualification and during incidents.
The API categories should also map to documentation and training. Operators should know which command retrieves current telemetry, which command retrieves recent events, which command clears history, and which command changes behavior.
