# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Firmware management is part of reliability because telemetry is only useful when deployed optics can be maintained safely. The spec points to the C M I S firmware management flow for local operations: image download, activation, and status monitoring.

It also describes a vendor-defined remote firmware upgrade protocol within the firmware management command group. That remote path uses in-band communication between the local and opposite ends.

The source flow diagram is dense, which is normal for a specification. The learner takeaway is the lifecycle pattern. Discover capability. Start the update. Transfer the image. Complete the download. Run the image. Commit it as the future default, or abort if the process fails.
Firmware workflows also need auditability. Operators should know what image was discovered, what was transferred, what status was returned, and whether activation or commit actually completed before declaring the update successful.
