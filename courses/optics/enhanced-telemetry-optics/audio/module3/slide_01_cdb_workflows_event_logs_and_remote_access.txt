# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Module three moves from what is measured to how the host gets it. The spec uses C D B, or Command Data Block, for command-reply exchange between host and module.

We will keep the level conceptual. You need to understand the prepare, trigger, poll, and retrieve workflow. You also need to understand why status checking, event logs, remote C M I S access, and payload handling matter for reliable operations.
If module two was the vocabulary, this module is the grammar. C D B defines how requests and replies are structured so host software can ask for data repeatedly and interpret the result with confidence.
