# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Remote C M I S access extends the same model to the far endpoint. The host does not open a separate direct management path into the remote module. It asks the local compliant module, and the local module retrieves the remote data in-band.

Conceptually, there are two useful access modes. Byte access is targeted. It is appropriate when software needs a small field or narrow range. Page access is broader. It retrieves a larger page-oriented snapshot.

This is powerful for operations because it gives the switch host a way to inspect both ends of the optical link. The operator can compare local and remote evidence without physically accessing the remote endpoint first.
Remote access also helps when physical access is slow or unavailable. Before dispatching someone to a rack or replacing a module, the operator can collect far-end evidence and decide whether the action is justified.
