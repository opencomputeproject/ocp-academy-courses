# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Context metrics are easy to underestimate. Temperature, voltage, received optical power, warning flags, and serial identity may not be the headline symptom, but they make the symptom traceable.

A module temperature value can help explain why a link degrades under a specific airflow or service condition. Laser temperature gives optical-device context. Received optical power helps separate path loss from a host electrical issue. D D M I warning and alarm flags tell you whether the module itself believes a threshold has been crossed.

Serial identity matters because event history follows physical hardware. If a module is moved, replaced, or compared against a qualification record, the telemetry needs to remain tied to the right object.
Context fields also help after hardware changes. If a module was reseated, swapped, or moved between ports, identity and environmental history can keep the investigation connected to the right physical device.
