# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
The metrics in this module are not just a long list. They are a way to turn link state into action.

Quality metrics help localize whether the weak signal is electrical, optical, line-side, host-side, or lane-specific. Error and recovery metrics show whether the weakness is causing correction load, relinks, or uncorrectable behavior. Environmental and identity fields make the evidence traceable.

The link-health score then compresses that evidence carefully. It uses the weakest category rather than an average, because a single bad field can still be the reason a link is unreliable. That idea prepares us for the workflow layer: how host software requests, checks, retrieves, and logs the data.
The practical test is whether the metric changes a decision. Does it point to optical cleaning, module replacement, firmware review, thermal inspection, or continued monitoring? Actionable telemetry narrows the next step.
