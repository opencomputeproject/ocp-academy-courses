# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Quality metrics help answer a location question. Is the signal margin degrading on the optical line side, or on the host-facing electrical side? The spec names S N R, eye height, PAM four level transition parameter, and M P I estimation among the values that must be available.

S N R describes signal relative to noise. Eye height gives another view of signal margin. PAM four level transition behavior helps characterize multilevel signaling. M P I estimation points toward optical path effects such as reflections or interference.

The operational value comes from separation. A line-side optical issue, a host-side electrical issue, and a single weak lane should not collapse into one vague port warning. The more precisely the metric is placed, the easier it is to choose the next action.
For troubleshooting, compare like with like. A line-side optical value should be compared against the optical path and remote partner behavior. A host-side electrical value should be compared against the local or remote electrical interface.
