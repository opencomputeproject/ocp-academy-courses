# pronunciation: CMIS -> C M I S; CDB -> C D B; I2C -> I squared C; SONiC -> sonic; PMD -> P M D; VDM -> V D M; DDMI -> D D M I; SNR -> S N R; PAM4 -> PAM four; MPI -> M P I; FEC -> feck; BER -> B E R; LOS -> L O S; LOL -> L O L; SDK -> S D K; API -> A P I
Module two turns the visibility model into measurements. The spec lists the telemetry that a compliant transceiver must make available to the host, either locally or from remote link partners over the in-band messaging interface.

We will group those fields into operational categories: optical quality, electrical quality, errors, recovery behavior, environmental context, and administrative identity. Then we will connect the categories to flags, masks, and the link-health score.
Think of this module as building a dashboard vocabulary. The values are technical, but the goal is operational: separate margin problems from service problems, and separate local symptoms from remote or segment-specific causes.
