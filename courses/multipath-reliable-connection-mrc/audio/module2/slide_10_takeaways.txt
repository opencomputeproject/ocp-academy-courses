# slide 2.10 — takeaways
# target: ~65s · ~155 words
# pronunciation: MRC -> "M R C"; RC -> "R C"; SRv6 -> "S R V six"; SACK -> "sack"; NACK -> "knack"; RDMA -> "R D M A"; PFC -> "P F C"

To recap module two. M R C is a thin extension to the R C transport. It keeps the verbs A P I and the rocky v two framing, supports only R D M A Write and Write-with-Immediate, and makes every data packet self-describing so the receiver places out-of-order arrivals straight to memory.

A single queue pair sprays its packets across many paths at once using entropy values, with three routing modes available: pure E C M P, structured entropy, and S R V six source routing with micro S I Ds.

Reliability sits on an independent sack and knack layer. Congestion control is N S C C from the Ultra Ethernet spec, augmented with packet trimming and E C N treated as a per-path signal.

And when a path fails, M R C parks that one entropy value in tens of microseconds — the connection survives, the job keeps moving, no P F C required.
