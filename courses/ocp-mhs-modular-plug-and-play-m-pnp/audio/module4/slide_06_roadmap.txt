# pronunciation: M-PNP -> "M P N P"; Fru -> "Fru" (read as the word, not letters); HPM -> "H P M"; FPGA -> "F P G A"; JSON -> "Jason"; BMC -> "B M C"; BIOS -> "BYE-oss"; LED -> "L E D"

The roadmap from here. Version zero point five shipped in September 2025 — that's the bundle we've spent this course in. Version zero point seven is targeted for August 2026. It will incorporate version zero point five community feedback, complete the remaining Fru, Discovery, and Boot specification sections, complete the HPM FPGA IP block definitions, fully define peripheral programmable logic device requirements, and update the JSON schemas. Version zero point nine is targeted around the OCP Global Summit in October 2026. By then, version zero point seven feedback will be incorporated and the Interoperability specification will be complete. Version one point zero is targeted for the first quarter of 2027. Between zero point five and one point zero, expect prototype work — JSON files for real platforms, FPGA IP block implementations, prototype BMC code and pseudocode — and ongoing chassis-level JSON definition work for things like LED policy and thermal management, which are still being scoped. The workstream is conservative: every version increment is gated on community feedback and prototype validation. If you want to influence what one point zero looks like, the time to engage is now, not after one point zero ships.
