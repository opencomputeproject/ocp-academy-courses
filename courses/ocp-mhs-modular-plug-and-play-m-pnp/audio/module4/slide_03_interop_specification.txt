# pronunciation: M-PNP -> "M P N P"; Fru -> "Fru" (read as the word, not letters); HPM -> "H P M"; FPGA -> "F P G A"; DC-SCM -> "D C S C M"; DMTF -> "D M T F"; PLDM -> "P L D M"; MCTP -> "M C T P"; MMBI -> "M M B I"; PCIe -> "P C I E"; SPDM -> "S P D M"

The Interoperability specification, version zero point three point zero, is short — fifteen pages — and that's intentional. It isn't trying to specify hardware. It's trying to define the contract that every M-PNP system must meet. It does five things. One: it states the vision and the definition of interoperability — what it means for two components from different vendors to work together mechanically, electrically, and logically. Two: it defines the three levels of interoperability — Plug-and-Code, Plug-and-Play Lite, and Baseline Plug-and-Play — so the community can talk about where any given platform sits. Three: it lays out the interoperability requirements that span specs. Four: it points to the conformance criteria. And five: it references every supporting design specification you need to implement against — the Fru, Discovery, and Boot spec, the HPM FPGA spec, DC-SCM two point one, HPM Common Circuits Type 1, OCP NIC three point zero, DMTF specs for FRU and PLDM and MCTP and MMBI, PCIe, and SPDM. The Interop spec is the table of contents for everything else. If you're new to M-PNP, read it first.
