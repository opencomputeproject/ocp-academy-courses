# pronunciation: Fru -> "Fru" (read as the word, not letters); frupack.py -> "Fru pack dot P Y"; fpga2json.py -> "F P G A to Jason dot P Y"; HPE -> "H P E"; FPGA -> "F P G A"; JSON -> "Jason"; EEPROM -> "E E PROM"; BMC -> "B M C"

Two tools today, more coming. The Fru Packaging Tool — frupack.py — was contributed to OCP by HPE and is live on GitHub at opencomputeproject slash OCP-SVR-MHS-M-PNP underscore FRUTool. It takes a JSON description and packages it into the binary Fru format an EEPROM expects, with the right multi-record framing, the right offsets, the right cyclic-redundancy checks. It's the tool you'll use to generate the bytes you actually program onto a board's Fru EEPROM. The FPGA-to-JSON tool — fpga2json.py — is the planned counterpart on the FPGA side. HPE has signaled it'll contribute that tool in the second quarter of 2026. The intent is that an FPGA vendor runs their build process, points fpga2json.py at the build output, and gets back the FPGA self-describing JSON we covered in Module 3. Other tools are needed too — JSON-from-schematic generators, JSON validators, BMC-side ingestion libraries. Some are in flight in the community; others are open opportunities for contributors. The OCP Innovation Stations at recent summits have included live demos of this tooling — worth a look if you're at a summit.
