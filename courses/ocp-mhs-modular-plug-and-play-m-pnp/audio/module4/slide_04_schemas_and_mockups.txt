# pronunciation: JSON -> "Jason"; HPM -> "H P M"; FPGA -> "F P G A"; UBM -> "U B M"; NVMe -> "N V M E"; M-XIO -> "M X I O"; PDB -> "P D B"

The bundle ships two JSON schemas and eight mockup examples. The hardware-component schema, version zero point five point six, describes any hardware component — HPMs, risers, backplanes, power distribution boards. The fpga schema, version zero point one point three, describes FPGAs and other programmable logic. Both schemas ship as raw JSON files plus PDF renderings of the schemas — open the PDFs first if you're new to JSON Schema. The mockup examples are the gold here. There's a complete Type 1 HPM mockup, version zero point six point eight, with 181 physical signals on the FPGA, every bus instance populated, and every connector described. Four riser mockups — single-slot x8, single-slot x16, two-slot x8, two-slot x16 — for the OCP M-XIO connector pinout. A UBM backplane mockup for NVMe drives. A Type 2 power distribution board mockup. And an FPGA-specific mockup at version zero point one point thirteen to pair with the HPM. Each ships with a PDF rendering. The mockups are how the workstream ships ground truth: read the JSON, compare to the spec, and you've got a worked example you can adapt for your own hardware.
