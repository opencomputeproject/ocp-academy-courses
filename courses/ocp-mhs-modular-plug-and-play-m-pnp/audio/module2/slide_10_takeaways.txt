# pronunciation: Fru -> "Fru" (read as the word, not letters); HPM -> "H P M"; BMC -> "B M C"; FPGA -> "F P G A"; JSON -> "Jason"; MMBI -> "M M B I"

Five things to take away. One: every component exposes Fru information in three layers — IPMI multi-record for identification, DMTF Fru for richer description with optional signing, and JSON for the deep self-description. Two: discovery is breadth-first across four phases, with each ring's JSON telling the BMC how to interpret the next ring. Three: security gates are baked into the discovery process — the FPGA is attested before it leaves reset, peripherals are attested before they leave Hold-Off. Four: the bundle ships concrete mockup JSON files for an HPM and six peripherals, so you can read the schema before you write any code. Five: BMC and Bios coordinate during early boot using MMBI, and the FPGA power state machine — the subject of Module 3 — is what makes the whole choreography possible.
