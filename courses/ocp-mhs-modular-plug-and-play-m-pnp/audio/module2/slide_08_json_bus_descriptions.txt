# pronunciation: JSON -> "Jason"; BMC -> "B M C"; I2C -> "I two C"; I3C -> "I three C"; JTAG -> "J TAG"; LTPI -> "L T P I"; M-PESTI -> "M PESTI"; PCIe -> "P C I E"; SGPIO -> "S G P I O"; UART -> "you art"; USB -> "U S B"

Every JSON file describes its component's physical world bus by bus. The Fru/Discovery/Boot spec defines a self-describing block for each one: I2C, I3C, JTAG, LTPI, M-PESTI, PCIe, SGPIO, UART, USB, and miscellaneous buses. Each block follows the same shape — bus instance, role, topology, devices on the bus — but with type-specific fields. The I2C block, for example, declares the multiplexer tree, the device addresses, the speed mode. The PCIe block declares lane assignments and bifurcation options. The M-PESTI block declares which tunneled signals run on which target. The connector block — separately — describes every physical connector on the board, which buses pass through it, and what's on the other side. The result, parsed by the BMC, is a directed graph of every signal in the platform. That graph is what makes plug-and-play tractable: the BMC isn't guessing at topology, it's reading it. And because every component publishes the same shape of JSON, one parser handles the whole system.
