# pronunciation: Fru -> "Fru" (read as the word, not letters); HPM -> "H P M"; BMC -> "B M C"; M-PNP -> "M P N P"

The 148 pages of this specification rest on three pillars. Pillar one: Fru information device requirements. Every component in the system — the HPM, every riser, every backplane, every power distribution board — exposes a Fru information device. That device holds a known, parsable record describing what the component is and where to find more detail. Pillar two: a deterministic, standardized discovery process. The BMC, starting from a known state, finds every component in a defined order, retrieves each one's Fru, and builds a complete topology of the platform before it lets the Bios run. Pillar three: boot process requirements. The boot sequence is interleaved with discovery — some attestation has to finish before main power is enabled, some before the operating system is allowed to run — and the spec lays out exactly which checkpoints must be hit at which power states. Hold those three pillars in mind. The next six slides each drill into a piece of one of them.
