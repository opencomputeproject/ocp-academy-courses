# pronunciation: BMC -> "B M C"; Bios -> "BYE-oss" (mixed case so it reads as a word); MMBI -> "M M B I"; DMTF -> "D M T F"; FPGA -> "F P G A"; PCIe -> "P C I E"; OOB -> "O O B"

Boot is where discovery meets the platform's actual purpose: getting an operating system to run. The boot process requirements lean on the FPGA power state machine — which we'll cover in detail in Module 3 — to gate progress. Some peripherals can't be discovered or attested until certain things have happened. PCIe devices, for example, can't be enumerated until the host SoC has executed enough of Bios to train the PCIe links. That's why the spec adds a power state called S0 Run-Bios-Boot — main power is on, PCIe training is underway, the BMC can attest devices over their out-of-band interface, but the OS is not yet allowed to take control. Only after the BMC has signed off on every peripheral that needed runtime attestation does the system transition to S0 Run-OS. Communication between BMC and Bios during early boot is required to use MMBI — DMTF's Memory-Mapped Buffer Interface, specification DSP0282 version one point zero. It's a request-and-response protocol over a shared memory region, and the spec requires both BMC and Bios to implement it. The specific payloads carried over MMBI are still being defined in the current revision, but the transport itself is locked.
