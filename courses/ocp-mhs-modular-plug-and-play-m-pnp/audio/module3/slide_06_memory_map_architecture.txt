# pronunciation: FPGA -> "F P G A"; BMC -> "B M C"; SPI -> "S P I"; LTPI -> "L T P I"; I2C -> "I two C"; I3C -> "I three C"; CSR -> "C S R"; CSRs -> "C S Rs"; OEM -> "O E M"; DC-SCM -> "D C S C M"

The FPGA memory map has a fixed shape. At the top of the map sits a required Memory Map Directory — a header that lists every section in the map and where to find it. Beneath the directory are required sections — the standardized IP blocks the spec mandates. Beneath those are optional but defined sections — the spec defines them but doesn't require them. And beneath those is an OEM section where vendors and integrators put their own content. The BMC reads the directory first; from the directory it learns where every other section starts and ends. Access into the memory map can come from three transports. SPI provides random-access read and write — the highest-bandwidth option. LTPI's data channel carries memory-mapped access multiplexed with virtual wires over the same link. I2C or I3C provide a mailbox-style access — slower, but always present on DC-SCM. Each access path has its own interface-specific CSRs at the top of the memory map. Below those CSRs is the directory. Below the directory is the rest. This layered structure means the BMC can use whichever transport is available in the current power state — including the I2C path that's required in standby — and still find the same content.
