# pronunciation: FPGA -> "F P G A"; HPM -> "H P M"; BMC -> "B M C"; MHS -> "M H S"; SGPIO -> "S G P I O"; SPI -> "S P I"; I2C -> "I two C"; I3C -> "I three C"; LTPI -> "L T P I"; OEM -> "O E M"; JSON -> "Jason"

Six takeaways. One: the FPGA standardization concept layers vendor logic, MHS-standardized logic, and integrator logic so each can change without breaking the others. Two: the FPGA must support virtual wires plus memory-mapped access, using SGPIO plus SPI or I2C-I3C, or LTPI for both. Three: thirteen IP blocks are defined; six are required. Four: the memory map is a directory plus required, optional, and OEM sections — accessible via SPI, LTPI data channel, or I2C-I3C mailbox. Five: the ten power states include four hold-off variants that exist specifically so the BMC can attest each tier before releasing reset. Six: the FPGA self-describing JSON tells the BMC exactly what's in this FPGA build, so one BMC firmware image can run any conforming HPM.
