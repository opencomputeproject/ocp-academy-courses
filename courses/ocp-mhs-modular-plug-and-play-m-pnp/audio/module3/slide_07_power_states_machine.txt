# pronunciation: FPGA -> "F P G A"; BMC -> "B M C"; SCM -> "S C M"; HPM -> "H P M"; DC-SCM -> "D C S C M"; Fru -> "Fru" (read as the word, not letters); SoC -> "S O C"; SoCs -> "S O Cs"; CSR -> "C S R"; PCIe -> "P C I E"; OOB -> "O O B"; Bios -> "BYE-oss" (mixed case so it reads as a word); RSMRST -> "R S M R S T"

This is the most important slide in this module. Ten power states. Five running, five hold-off variants where the device is powered but reset is held. The state machine starts at G3 — mechanical off, only coin-cell battery power. When the DC-SCM is seated, twelve-volt standby flows to the SCM, and the system transitions to Pre-Standby. The SCM BMC boots, performs HPM discovery, brings up HPM aux rails, and asserts HPM_STBY_EN. The system moves to Standby Hold-Off. At this point the FPGA is powered but held in reset. The BMC attests the FPGA. If attestation passes, the BMC de-asserts HPM_STBY_RST hash and the FPGA exits reset. Standby Run. From here on, the FPGA itself holds the center of truth for the power state. The BMC writes the next power-state transition into a CSR in the power management IP block. The FPGA performs the transition and raises a transition interrupt when it's done. Next stop: S5 Hold-Off. All aux rails are in regulation. The BMC attests the Fru storage devices. If those pass, the BMC writes Transition to S5 Run. RSMRST hash de-asserts. The system is in S5 Run, awaiting a power-on request. Power-on enables the twelve-volt primary rail, the system transitions to S0 SoC Ramp, the host SoC rails sequence up, and we land in S0 Hold-Off. Another attestation gate — this time the BMC attests the SoCs themselves. Then S0 Run-Bios-Boot, where PCIe training happens and the BMC attests PCIe devices over their out-of-band interface. Finally S0 Run-OS, when Bios hands control to the operating system. Three pauses, three attestation gates. Every gate is optional per the spec — security policy is up to the system integrator — but the state machine exists to make those gates possible without invasive workarounds.
