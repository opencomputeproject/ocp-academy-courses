# pronunciation: M-PNP -> "M P N P"; DC-MHS -> "D C M H S"; HPM -> "H P M"; BMC -> "B M C"; FPGA -> "F P G A"; Fru -> "Fru" (read as the word, not letters); JSON -> "Jason"

Three things to carry into the rest of the course. First: M-PNP is not new hardware. It's a way of describing existing hardware so that one BMC firmware image can run the whole DC-MHS ecosystem without custom enablement per HPM. Second: the unit of description is a JSON file the hardware vendor writes at design time. The BMC retrieves it at boot, parses it, and gains complete knowledge of the board. Third: the specifications bundle is layered. The Interoperability specification sets the contract. The Fru, Discovery, and Boot specification operationalizes it. The HPM FPGA specification gives the BMC a stable surface to talk to, so discovery actually works. Keep that three-part structure in mind. The next two modules are organized around it.
