# pronunciation: M-PNP -> "M P N P"; DC-MHS -> "D C M H S"; DC-SCM -> "D C S C M"; HPM -> "H P M"; M-FLW -> "M F L W"; M-DNO -> "M D N O"; M-CRPS -> "M C R P S"; M-PIC -> "M PIC"; M-XIO -> "M X I O"; M-PESTI -> "M PESTI"; M-SIF -> "M S I F"; LTPI -> "L T P I"; SGPIO -> "S G P I O"; FPGA -> "F P G A"; BMC -> "B M C"

M-PNP doesn't live alone. It's a workstream inside DC-MHS — the Datacenter Modular Hardware System project — and it sits on top of a stack of sibling specifications that define the physical and electrical layers. M-FLW and M-DNO define the full-width and density-optimized HPM form factors. M-CRPS defines the common redundant power supply interface. M-PIC handles platform infrastructure connectivity — power, control panel, cooling. M-XIO defines the high-speed extensible I/O connector and pinout strategy. M-PESTI defines the peripheral sideband tunneling interface used by risers and backplanes to identify themselves. M-SIF covers shared infrastructure. Underneath all of that, DC-SCM two point one specifies the secure control module that hosts the BMC, and HPM Common Circuits Type 1 specifies the standardized circuits the SCM expects on the HPM side. M-PNP plugs into all of these. The Fru, Discovery, and Boot specification consumes M-PESTI for peripheral identification. The HPM FPGA specification consumes LTPI, SGPIO, M-XIO, and M-PIC as transport. And the Interoperability specification references every one of them as conformance criteria. M-PNP is the discovery and interoperability layer that turns this stack of hardware specs into something a single firmware image can run on.
