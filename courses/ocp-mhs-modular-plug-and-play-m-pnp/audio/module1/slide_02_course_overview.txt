# pronunciation: M-PNP -> "M P N P"; Fru -> "Fru" (read as the word, not letters); FPGA -> "F P G A"; HPM -> "H P M"; BMC -> "B M C"; Bios -> "BYE-oss" (mixed case so it reads as a word)

Welcome to OCP Modular Plug-and-Play. Four modules to take you from the motivation through the technical core, and on to the on-ramp for contributing back. Module one sets context: the problem and the vision. Module two takes apart the Fru, Discovery, and Boot specification — how a BMC learns the platform from a cold start. Module three covers the H P M F P G A specification — the standardized API surface every HPM FPGA exposes to the BMC, including the power state machine with attestation hold-offs. Module four pulls it together with the Interoperability specification, the tooling, the roadmap from version zero point five to one point zero, and how you can join the workstream. This course is built for hardware engineers, BMC and Bios firmware developers, FPGA designers, and system architects implementing against the M-PNP specifications bundle version zero point five.
