# pronunciation: M-PNP -> "M P N P"; Fru -> "Fru" (read as the word, not letters); HPM -> "H P M"; FPGA -> "F P G A"; UBM -> "U B M"; JSON -> "Jason"; PDB -> "P D B"

The M-PNP specifications bundle is what gets released. Three normative specifications and a set of supporting JSON files. The Interoperability specification — currently at version zero point three point zero — is the glue. It defines what interoperability means in this context, the conformance levels, and which other specs you must reference. The Fru, Discovery, and Boot specification — version zero point five point zero RC2, 148 pages — is the operational core. It tells the BMC where to look, what to read, what to authenticate, and in what order to bring the system up. The HPM FPGA specification — version zero point five point zero RC2, 227 pages — defines the standardized FPGA architecture that makes the discovery process tractable in the first place. Alongside the specifications ship two JSON schemas. The hardware-component schema, version zero point five point six, describes HPMs and peripherals. The fpga schema, version zero point one point three, describes the FPGA itself. And there are JSON mockup examples: a complete Type 1 HPM, four riser variants, a UBM backplane, and a Type 2 power distribution board — each shipped with both the raw JSON and a PDF rendering so you can read the structure on paper before you parse it in code.
