# pronunciation: RBT -> R B T; MCTP -> M C T P; NC-SI -> N C S I; PCIe -> P C I E

The specification defines three management interface types, and every compliant card must support at least one. The preferred option is R B T plus M C T P. That combination uses N C S I over R B T for real-time Nick control and network pass-through, while M C T P handles platform management functions such as discovery, monitoring, and firmware update.

The second type is R B T, which provides N C S I over an R M I I physical sideband channel separate from host P C I E. The third type is M C T P. M C T P is not limited to S M Bus; the referenced D M T F specifications include bindings such as S M Bus or I squared C, P C I E V D M, and USB. This is why the transport choice matters when reading a card's capability declarations.
