# pronunciation: NC-SI -> N C S I; RBT -> R B T; RBT_ISOLATE -> R B T isolate

N C S I, the Network Controller Sideband Interface, is how the B M C can control the Nick and use its network port without depending on the host CPU. Control traffic configures network parameters, queries link state, and manages channels. Pass-through traffic lets the B M C send and receive packets through the Nick Ethernet port for functions such as I P M I over LAN and Redfish.

In multi-slot systems, token-based arbitration prevents multiple slots from driving the bus at the same time. R B T isolate is equally important: it prevents a powered-down or isolated slot from pulling the shared bus low. Do not describe R B T hot swap as a fully specified feature; the spec gives the interface behavior and isolation rules, while complete platform hot-swap design stays outside this scope.
