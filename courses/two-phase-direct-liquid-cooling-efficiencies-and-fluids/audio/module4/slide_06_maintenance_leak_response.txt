# pronunciation: SDS -> "S D S"; TCS -> "T C S"
Maintenance and leak response look different when the server-side fluid is dielectric.

The OCP guideline says the dielectric fluid is not expected to promote biological growth, corrosion, or residue formation. Because of that, the system typically needs only minimal filtration, around one hundred microns, during normal operation. Additional filtration is still recommended during loading or cleaning so introduced particles do not become a system problem.

Leak response also changes. The guideline states that because the dielectric liquid is non-conductive and poses no risk to the server or its components, there is no immediate requirement to shut down the server or power off components when a leak occurs.

That does not mean leaks are ignored. The source must be located. A standard refrigerant sniffer can help pinpoint the leakage area. Buffer-level changes can also indicate liquid loss.

That distinction is important: first-drop detection is not the same kind of emergency as it would be with conductive liquid inside electronics. The operator still needs to detect the leak, find it, fix it, and manage the fluid responsibly.

The balanced message is important. Dielectric fluid reduces some immediate IT risk, but it does not remove the need for detection, documentation, S D S guidance, repair procedures, and lifecycle accountability.
