The "ee-sun" Header is exactly four bytes — thirty-two bits. Let's walk them. Rev is two bits, a revision number that allows future versions of the header without breaking what's already deployed. For version one point zero, Rev is zero zero. Then comes F, a single bit — the Flow Label Valid flag. When F is set, the Flow Label later in the header carries usable entropy, and switches must use it for deterministic hashing. Next is EH-Co S, three bits — the "ee-sun" class of service. This replaces the eight-oh-two dot one p priority field at L two and the D S C P bits inside the old I P TOS. Three bits gives eight traffic classes, which is plenty for a scale-up domain that doesn't run V LANs. When both the eight-oh-two dot one Q tag and CBFC carry their own class of service, EH-Co S takes precedence — that's spec-level. Then EH-E C N, two bits — the explicit congestion notification field, encoded exactly the way E C N is encoded in I P TOS today. Senders signal that they're E C N capable; switches signal that they experienced congestion; the transport at the endpoint responds. The big block in the middle is the Flow Label, sixteen bits — a source-controlled value that the fabric uses for load-balancing entropy alongside the destination and source addresses. Then T T L, four bits — Time To Live. Every switch decrements it on forwarding, and a packet whose T T L reaches zero is dropped. That's how "ee-sun" avoids loops in a hierarchical topology that doesn't have an I P header to lean on. Two more bits are UD — User Defined — populated and consumed end-to-end by the transport at the endpoints. The final two bits are reserved for future use. One last detail: any switch that modifies the header in flight — for example, marking E C N or decrementing T T L — must recompute the frame check sequence.
