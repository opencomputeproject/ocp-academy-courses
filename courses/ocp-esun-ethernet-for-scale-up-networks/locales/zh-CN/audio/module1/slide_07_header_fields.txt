ESUN Header 正好是4字节，也就是32位。首先是2位的 Rev，用于标识修订版本，v1.0 中为00。接下来是1位的 F，也就是 Flow Label 有效标志。F 设置时，交换机必须使用后面的 Flow Label 进行确定性哈希。EH-CoS 是3位的服务类别字段，替代 L2 的802.1p 优先级和原 IP TOS 中的 DSCP。3位可以表示8个流量类别。当802.1Q 标签和 CBFC 也携带服务类别时，规范规定 EH-CoS 优先。EH-ECN 是2位的显式拥塞通知字段，编码方式与 IP ECN 相同。中间的 Flow Label 为16位，由源端设置，为负载均衡提供额外哈希熵。TTL 为4位，每台交换机转发时将其递减，到0的数据包被丢弃，从而在没有 IP Header 的分层拓扑中防止环路。UD 是2位的用户定义字段，由端点之间的传输层使用，最后2位保留。交换机如果在传输过程中标记 ECN 或递减 TTL，就必须重新计算帧校验序列 FCS。
