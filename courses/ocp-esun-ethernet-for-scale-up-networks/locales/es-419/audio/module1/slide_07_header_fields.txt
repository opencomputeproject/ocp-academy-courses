El Encabezado ESUN tiene exactamente cuatro bytes, o treinta y dos bits. El primer campo es Rev, con dos bits para identificar la revisión. En la versión 1.0, Rev es cero cero. Luego aparece F, un bit que indica si el Flow Label es válido. Cuando F está establecido, el Flow Label aporta entropía utilizable y los switches deben incluirlo en el hash determinista. EH-CoS tiene tres bits y representa la clase de servicio ESUN. Reemplaza la prioridad 802.1p de L2 y los bits DSCP del antiguo IP TOS. Tres bits ofrecen ocho clases de tráfico. Cuando una etiqueta 802.1Q y CBFC también llevan una clase de servicio, EH-CoS tiene precedencia. EH-ECN tiene dos bits y usa la misma codificación de notificación explícita de congestión que IP ECN. El Flow Label ocupa dieciséis bits y agrega entropía para balanceo de carga junto con las direcciones de destino y origen. TTL tiene cuatro bits. Cada switch disminuye el valor y descarta el paquete cuando llega a cero, evitando bucles sin depender de un Encabezado IP. UD tiene dos bits de uso de extremo a extremo y los dos bits finales están reservados. Todo switch que marque ECN o disminuya TTL debe recalcular el FCS.
