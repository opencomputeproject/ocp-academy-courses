ESUN Header 正好是4位元組，也就是32位。首先是2位的 Rev，用於標識修訂版本，v1.0 中為00。接下來是1位的 F，也就是 Flow Label 有效標誌。F 設定時，交換機必須使用後面的 Flow Label 進行確定性雜湊。EH-CoS 是3位的服務類別欄位，替代 L2 的802.1p 優先順序和原 IP TOS 中的 DSCP。3位可以表示8個流量類別。當802.1Q 標籤和 CBFC 也攜帶服務類別時，規範規定 EH-CoS 優先。EH-ECN 是2位的顯式擁塞通知欄位，編碼方式與 IP ECN 相同。中間的 Flow Label 為16位，由源端設定，為負載均衡提供額外雜湊熵。TTL 為4位，每臺交換機轉發時將其遞減，到0的資料包被丟棄，從而在沒有 IP Header 的分層拓撲中防止環路。UD 是2位的使用者定義欄位，由端點之間的傳輸層使用，最後2位保留。交換機如果在傳輸過程中標記 ECN 或遞減 TTL，就必須重新計算幀校驗序列 FCS。
